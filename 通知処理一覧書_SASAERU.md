# SASAERU 通知処理タイミングと内容 一覧書

**バージョン:** 1.0  
**作成日:** 2026-06-20  
**対象システム:** SASAERU v2.0

---

## 通知インフラ概要

| 手段 | 送受信方向 | 実装 |
|-----|-----------|-----|
| **管理者メール** | システム → 管理者 | Nodemailer + Netlify Function (`/.netlify/functions/send-email`) |
| **Slack通知** | システム → 管理者チャンネル | Slack Incoming Webhook（`send-email` Function 内で実施） |
| **クラブオーナーメール** | システム → クラブ担当者 | Nodemailer + Netlify Function (`send-email`) |
| **申請者・問い合わせ者メール** | システム → 企業担当者 | Nodemailer + Netlify Function (`send-email`) |
| **メールアドレス確認メール** | システム → 新規登録ユーザー | Netlify Function (`/.netlify/functions/send-verify-email`) |

---

## 通知一覧

### No.1 ── クラブ登録申請（新規）

| 項目 | 内容 |
|-----|-----|
| **発火タイミング** | ユーザーが `index.html` の登録フォームを送信し、Firebase Auth + Firestore 書き込みが完了した直後 |
| **発火元ファイル** | `index.html` → `sendAdminNotification('club_apply', ...)` |
| **通知タイプ** | `admin_notify`（管理者宛） |
| **送信先** | 管理者メール・管理者Slackチャンネル |
| **通知ラベル** | `🏃 新規クラブ申請` |
| **通知内容（パラメータ）** | クラブ名・カテゴリ・エリア・メールアドレス |
| **条件** | 無条件（登録成功時は必ず送信） |

---

### No.2 ── メールアドレス確認メール送信

| 項目 | 内容 |
|-----|-----|
| **発火タイミング** | クラブ登録申請完了直後（Firebase Auth アカウント作成・Firestore 保存の直後） |
| **発火元ファイル** | `index.html` → `fetch('/.netlify/functions/send-verify-email', ...)` |
| **通知タイプ** | `send-verify-email` Function |
| **送信先** | 新規登録ユーザー（クラブ担当者）のメールアドレス |
| **通知内容** | メールアドレス確認リンク付きメール |
| **条件** | 無条件（登録成功時は必ず送信） |
| **備考** | マイページ（`mypage.html`）からも再送信ボタンで同じ Function を呼び出し可能 |

---

### No.4 ── お問い合わせ送信

| **条件** | 無条件（送信成功時は必ず送信） |

### No.5 ── 支援申請（企業 → クラブ）

| 項目 | 内容 |
|-----|-----|
| **発火タイミング** | `club.html` の支援申請フォーム送信・Firestore `inquiries` 保存完了直後 |
| **発火元ファイル** | `club.html` → `sendAdminNotification('inquiry', ...)` |
| **通知タイプ** | `admin_notify`（管理者宛） |
| **送信先** | 管理者メール・管理者Slackチャンネル |
| **通知ラベル** | `📩 支援申請` |
| **通知内容（パラメータ）** | クラブ名・企業名・担当者名・メールアドレス・内容（先頭100文字） |
| **条件** | 無条件（送信成功時は必ず送信） |

---

### No.6 ── クラブオーナーへの支援申請通知

| 項目 | 内容 |
|-----|-----|
| **発火タイミング** | 支援申請（No.5）と同タイミング（並列送信） |
| **発火元ファイル** | `club.html` → `fetch('send-email', { type: 'inquiry_owner', ... })` |
| **通知タイプ** | `inquiry_owner` |
| **送信先** | 対象クラブの `contact_email` |
| **通知内容（パラメータ）** | `toEmail`・`orgName`・申請企業名・担当者・支援内容など |
| **条件** | `currentOrgContactEmail` が存在する場合のみ送信 |

---

### No.7 ── 申請企業への自動返信メール

| 項目 | 内容 |
|-----|-----|
| **発火タイミング** | 支援申請（No.5）と同タイミング（並列送信） |
| **発火元ファイル** | `club.html` → `fetch('send-email', { type: 'inquiry_reply', ... })` |
| **通知タイプ** | `inquiry_reply` |
| **送信先** | 申請企業担当者のメールアドレス（`data.sender_email`） |
| **通知内容（パラメータ）** | `toEmail`・`companyName`・クラブ名など |
| **条件** | 無条件（送信成功時は必ず送信） |

---

### No.8 ── 審査承認通知（クラブへ）

| 項目 | 内容 |
|-----|-----|
| **発火タイミング** | 管理者が `admin.html` で「承認・公開」ボタンをクリックし、Firestore `status → public` 更新完了直後 |
| **発火元ファイル** | `admin.html` → `sendOrgStatusEmail('approve', ...)` |
| **通知タイプ** | `approve`（Function 内でテンプレート使い分け） |
| **送信先** | クラブの `contact_email` |
| **通知内容（パラメータ）** | `toEmail`・`toName`・`clubName` |
| **条件** | 管理者による承認操作時のみ |
| **失敗時の動作** | `showToast('承認メールの送信に失敗しました。手動でご連絡ください。', false)` |

---

### No.9 ── 審査却下通知（クラブへ）

| 項目 | 内容 |
|-----|-----|
| **発火タイミング** | 管理者が `admin.html` で「却下」ボタンをクリックし、Firestore `status → rejected` 更新完了直後 |
| **発火元ファイル** | `admin.html` → `sendOrgStatusEmail('reject', orgData, reason)` |
| **通知タイプ** | `reject` |
| **送信先** | クラブの `contact_email` |
| **通知内容（パラメータ）** | `toEmail`・`toName`・`clubName`・`rejectReason`（却下理由） |
| **条件** | 管理者による却下操作時のみ |
| **失敗時の動作** | `showToast('却下メールの送信に失敗しました。手動でご連絡ください。', false)` |

---

### No.10 ── 新規支援メニュー登録（管理者通知）

| 項目 | 内容 |
|-----|-----|

| **発火元ファイル** | `mypage.html` → `fetch('send-email', { type: 'admin_notify', label: '🆕 新規支援メニュー登録', ... })` |
| **通知タイプ** | `admin_notify`（管理者宛） |
| **送信先** | 管理者メール・管理者Slackチャンネル |
| **通知ラベル** | `🆕 新規支援メニュー登録` |
| **通知内容（パラメータ）** | 団体名・メニュー名・希望金額/数量（任意） |
| **条件** | 無条件（新規保存時は必ず送信） |

---

### No.11 ── 新規支援メニュー登録（クラブへの完了通知）

| 項目 | 内容 |
|-----|-----|
| **発火タイミング** | 支援メニュー新規保存（No.10）と同タイミング |
| **発火元ファイル** | `mypage.html` → `fetch('send-email', { type: 'menu_created', ... })` |
| **通知タイプ** | `menu_created` |
| **送信先** | クラブの `contact_email` |
| **通知内容（パラメータ）** | `toEmail`・`orgName`・`menuTitle` |
| **条件** | `currentOrg.status === 'public'` かつ `currentOrg.contact_email` が存在する場合のみ |

### No.12 ── 支援マッチング成立（管理者通知）

| 項目 | 内容 |

| **通知タイプ** | `admin_notify`（管理者宛） |
| **送信先** | 管理者メール・管理者Slackチャンネル |
| **通知ラベル** | `🏆 支援マッチング成立` |
| **通知内容（パラメータ）** | 団体名・メニュー名・継続表示フラグ（募集継続か終了か） |
| **条件** | 無条件（マッチング成立操作時は必ず送信） |

## 通知タイプ（Function typeパラメータ）一覧

| type値 | 説明 | 送信先 |
|-------|-----|-------|
| `admin_notify` | 管理者通知（汎用） | 管理者 + Slack |
| `applicant` | クラブ登録申請者への自動返信 | 申請クラブ |
| `inquiry_owner` | 支援申請受信をクラブへ通知 | クラブオーナー |
| `inquiry_reply` | 支援申請企業への自動返信 | 申請企業担当者 |
| `approve` | 審査承認をクラブへ通知 | クラブ担当者 |
| `reject` | 審査却下をクラブへ通知 | クラブ担当者 |
| `menu_created` | 支援メニュー登録完了通知 | クラブ担当者 |
| ─（専用Function） | メールアドレス確認 | 登録ユーザー |
